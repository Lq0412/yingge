# 项目 MVP 说明

## 1. 概述
聚焦“英歌舞服装在线试装”最小闭环：注册/登录 → 上传图片 → 提交试装任务 → 生成结果 → 查看记录。

## 2. 目标
- 验证试装可行性与用户兴趣。
- 打通上传 → 生成 → 记录的主流程。
- 为后续扩展（商家/活动接入、分享等）预留结构。

## 3. MVP 范围
### 3.1 账号与鉴权
- 注册/登录（手机或邮箱 + 密码）；Session 鉴权。

### 3.2 图片上传
- 上传人像/服装图（jpg/png、≤ 5MB），存 COS，DB 仅存 URL/类型/归属用户。

### 3.3 服装素材
- 内置/预置示例英歌舞服装；后续自助上传/标签管理延后。

### 3.4 试装生成（异步）
- 输入人像 URL + 服装 URL + 可选 prompt。
- 后端异步调用火山 Ark SDK 生成结果，下载并上传 COS，写入任务/记录。
- 任务状态：pending/processing/done/failed，轮询获取。

### 3.5 试装记录
- 字段：用户、三张图 URL、状态、错误信息、时间；列表/详情，仅本人访问；暂不删除。

### 3.6 结果查看
- 返回 COS 结果 URL，可下载；分享链路延后。

## 4. 非功能 & 验收
- 性能：生成 ≤ 10s；普通接口 p95 ≤ 1s。
- 稳定性：外部 AI/COS 异常有明确错误；关键路径记录日志。
- 安全：密码加密；鉴权后访问个人资源；图片仅用于试装。
- 验收：完成一次完整试装并记录可查；异常路径有合理提示。

## 5. 技术栈与依赖
- 后端：Spring Boot 3.5.x + Java 21，MyBatis-Plus，异步线程池。
- 存储：MySQL；对象存储 COS。
- AI：火山 Ark SDK（试装）；Seedream 预留。
- 文档：Knife4j/Swagger。
