# 项目 MVP 说明（英歌舞在线试装）

## 1. 概述
MVP 聚焦“英歌舞服装在线试装”最小可用闭环：用户完成注册/登录，上传个人照片，选择预置的英歌舞服装，触发 AI 生成效果图，查看结果与历史记录。

## 2. 目标
- 用 1 套最小功能验证可行性与用户兴趣。
- 控制接入复杂度，优先打通上传 → 生成 → 查看记录。
- 为后续商家/活动接入预留扩展点（API 结构、数据模型可扩展）。

## 3. MVP 范围
### 3.1 账号与鉴权
- 注册/登录（手机或邮箱 + 密码）。
- 会话维持：JWT 或 Session；接口鉴权统一校验。

### 3.2 图片上传与存储
- 支持上传 1 张人像 + 服装图（服装图 MVP 可用内置素材）。
- 文件校验：jpg/png、≤ 5MB。
- 存储到对象存储，数据库仅存 URL、类型、归属用户。

### 3.3 服装库
- 内置一批英歌舞服装示例数据（通过运营导入或脚本预置）。
- 商家自助上传与标签管理不在 MVP 范围。

### 3.4 AI 试装
- 前端选择人像图与服装图后发起试装请求。
- 后端调用外部 AI 服务生成结果，同步返回 URL；失败需有明确错误提示。
- 结果图上传对象存储并写入试装记录。

### 3.5 试装记录
- 记录字段：用户、三张图片 URL、状态、错误信息、创建时间。
- 列表（倒序）与详情接口；MVP 暂不做删除。

### 3.6 结果查看与下载
- 前端展示结果图，提供下载；分享链路延后。
- 结果链接需具备基础防护（鉴权或临时签名）。

## 4. 非功能 & 验收
- 性能：单次生成 ≤ 10 秒；普通接口 p95 ≤ 1 秒。
- 稳定性：AI/COS 异常返回明确错误；关键路径记录日志。
- 安全：密码加密存储；鉴权后访问个人资源；图片仅用于本服务。
- 验收：能完成 1 次完整试装并在记录中查看；异常路径（无登录/图片超限/AI 失败）均有合理提示。

## 5. 技术栈与依赖
- 后端：Java + Spring Boot，RESTful API，JPA/Mapper 访问数据库。
- 存储：MySQL（或等价关系型 DB）；对象存储 COS/OSS。
- AI：Seedream 或等价试装 API。
- 文档：Swagger/Knife4j 暴露接口描述。

## 6. 版本外内容（延后）
- 商家后台（素材管理、数据看板）。
- 结果分享/社交、批量生成、重试与多样性增强。
- 短信/邮件验证码登录、隐私合规的图片删除/过期策略（可在下一版实现）。
